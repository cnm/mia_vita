HOWTO: Get crisp lines with no anti-aliasing

By default HTML5 canvas anti-aliases lines that you draw and this can make them appear to be thicker than they really are. You can avoid this if you prefer and get clean crips lines by offsetting your coordinates to the nearest half pixel. To make this easier there is a funtion in RGraph that will determine the nearest half-pixel value and change your values for you.

1. The original un-altered line

[No canvas support]

Here is the original line drawn normally without using the RGraph AA() function 


<script>
    window.onload = function ()
    {
        var canvas  = document.getElementById("cvs");
        var context = canvas.getContext('2d');
        
        context.beginPath();
            context.moveTo(125, 20);
            context.lineTo(125, 230);
        context.stroke();
    }
</script>

2. The line after the anti-aliasing fix has been applied

[No canvas support]

And here is the line after the half-pixel offset has been applied (the right-hand line). The line on the left is the original un-altered line so that you can easily compare. Note that here, because there is no RGraph object the values have been changed manually, though if you're using an RGraph object there is a special function that you can use: AA(obj, value) where obj is the RGraph chart object and value is your value that you want to change. 

<script>
    context.moveTo(AA(obj, 125), 20);
    context.lineTo(AA(obj, 125), 230);
</script>
